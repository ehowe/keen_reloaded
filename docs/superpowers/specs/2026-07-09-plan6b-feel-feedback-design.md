# Plan 6b — Feel & Feedback — Design Spec

**Date:** 2026-07-09
**Status:** Implemented (merged to `main`; 362/362 GUT green)
**Parent spec:** `docs/superpowers/specs/2026-06-25-keen-reloaded-design.md` (§6 Game Runtime)
**Engine:** Godot 4.7 (stable), GDScript
**Predecessors:** Plans 1–5, Plan 6a (audio)
**Scope owner:** runtime + core + ui

## 1. Overview

Plan 6 (Polish) was decomposed into three independent sub-plans:

| Sub-plan | Areas | Status |
|---|---|---|
| 6a — Audio | music + SFX | ✅ done |
| **6b — Feel & Feedback** | **HUD polish + gamepad mapping** | **this spec** |
| 6c — Persistence | save / progression (disk) | done |

Plan 6b delivers two independent improvements, bundled because both are small and
both are "feel" surface area:

1. **Icon-based HUD.** Replace the single unstyled `"Score: X   Ammo: Y   HP: Z"`
   `Label` (built in `LevelRuntime._build_hud`) with a dedicated `Hud` scene:
   health as heart icons, ammo as raygun-shot icons, score as a styled number.
   Add a minimal overworld HUD showing completed-level progress
   (`Levels cleared: N / M`).
2. **Default gamepad bindings.** Attach `InputEventJoypadMotion` / `InputEventJoypadButton`
   events to the 8 existing input actions (move_*, jump, pogo, shoot, interact),
   alongside the existing keyboard bindings. No remap UI, no settings persistence.

Icon assets are **generated placeholder PNGs** (synthesized by a one-off headless
tool, mirroring `tools/gen_audio.gd`), committed under `assets/sprites/hud/`, and
free to replace later with real Keen art.

```
Boot
  GameManager._ready -> _ensure_input_actions()
    -> each action gets keyboard + joypad events (idempotent)

LevelRuntime.build(level)
  -> _spawn_player -> _build_hud(player)
       -> instantiate hud.tscn into a CanvasLayer (layer=1)
       -> LEVEL mode:   hearts/ammo/score, wired to player signals
       -> OVERWORLD mode: "Levels cleared: N / M" (N=completed_levels.size(),
                                                   M=entrance entities on overworld)

tools/gen_hud_icons.gd  (one-off, headless)
  -> assets/sprites/hud/{heart_full,heart_empty,ammo_full,ammo_empty}.png
```

### Goals

| # | Goal |
|---|------|
| 1 | In-level HUD shows health as heart icons, ammo as shot icons, and score as a styled number, updating live from player signals. |
| 2 | Overworld HUD shows `Levels cleared: N / M`, where M counts the entrances on that overworld and N is `GameManager.completed_levels.size()`. |
| 3 | A dedicated, isolated `Hud` scene is instantiated by `LevelRuntime` (mirrors `completion_overlay` pattern); the runtime no longer builds HUD widgets inline. |
| 4 | The 8 input actions each accept gamepad input (D-pad + left stick for movement; A/B/X/Y for jump/pogo/shoot/interact) in addition to the existing keyboard bindings. |
| 5 | Four CC0 placeholder icon PNGs are generated by a committed tool and committed under `assets/sprites/hud/`; replaceable later. |
| 6 | GUT tests verify HUD widget state, mode switching, gamepad event attachment, and runtime wiring — headless and deterministic (no visual-output assertion). |

### Out of scope

- **Game-feel / juice** — screen shake, hit-flash, landing squash, particle puffs. Descoped; "feel" here means input+HUD surface only.
- **Remappable controls / settings UI / persisted bindings.** (Possible future plan; touches save = 6c territory.)
- **Per-bus volume / mute UI.** (6a left this out.)
- **Sprite font** (`Lettering.png`) — HUD uses the default Godot font with theme overrides.
- **In-level timer / level-name header** in the HUD. `elapsed` stays on the completion screen only.
- **Interact-seam cleanup** — `interact` keeps its existing keyboard (Up) binding; we only *add* Y on gamepad. Generalizing the hardcoded Up is deferred.
- **Save / progression** — sub-plan 6c.

## 2. Key decisions

| Decision | Choice | Rationale |
|---|---|---|
| HUD ownership | **Dedicated `Hud` scene instantiated by `LevelRuntime`** | One responsibility, testable in isolation (instantiate + call setters), mirrors the existing `completion_overlay` pattern. Keeps `LevelRuntime` focused on orchestration. |
| Health display | **Heart icons, one slot per `max_health`** | Matches Keen's life-icon convention; makes damage visible at a glance. Requires adding `Player.max_health` (default 3, matching today's fixed health). |
| Ammo display | **Shot icons, one slot per `max_ammo`** | Symmetric with hearts; `max_ammo` already exists on `Player`. Filled slot = current ammo, empty = remaining capacity. |
| Icon sourcing | **Procedural generator → committed CC0 PNGs** | Matches the `gen_audio` pattern: deterministic, reproducible, no hand-authored art, trivially replaceable. Avoids vector `_draw()` entangling icon rendering with the HUD node. |
| Overworld "M" (total) | **Count of `level.entities` whose `properties.target_level_id != ""`** | Episode-agnostic (entrances are identified by carrying a `target_level_id`, not by a namespaced type_id string). Computable from `LevelData` at HUD-build time — no need to wait for entity spawn. |
| Overworld "N" (cleared) | **`GameManager.completed_levels.size()`** | The existing session-held completion set; cleared on episode start via `clear_progress()`, so no stale cross-episode bleed. |
| Gamepad binding style | **Additive events on existing actions** | Godot `InputMap` actions accept multiple events; keyboard and gamepad coexist. No new actions, no action renames, no input-action generalization pass. |
| Gamepad movement | **D-pad buttons + left stick axis** | Both are conventional; supporting both costs nothing and suits players with either preference. Axis events use `deadzone = 0.5`. |
| Idempotency | **Keep `if InputMap.has_action(action): return` guard** | `_ensure_input_actions` may run once at boot; the guard prevents duplicate events on re-entry. Since `project.godot` declares no `[input]` section, actions are always created fresh on first run. |
| HUD layering | **HUD `CanvasLayer` layer = 1; completion overlay `CanvasLayer` layer = 10** | Ensures the completion panel sits above the HUD when a level is cleared (the panel is centered, not full-screen, so the HUD would otherwise show through). |

## 3. HUD component

**Files:** `src/ui/hud.gd`, `src/ui/hud.tscn`

### 3.1 Node tree

```
Hud (Control; anchors = top-left; mouse_filter = IGNORE)
  LevelContainer (VBoxContainer; visible when mode == LEVEL)
    HeartsRow   (HBoxContainer)   # children added dynamically
    AmmoRow     (HBoxContainer)   # children added dynamically
    ScoreLabel  (Label)           # "Score 1234"
  OverworldContainer (VBoxContainer; visible when mode == OVERWORLD)
    ClearedLabel (Label)          # "Levels cleared: 2 / 5"
```

`Hud` is anchored top-left with a small margin (e.g. x=12, y=8, matching the old
label position). `mouse_filter = IGNORE` so it never intercepts input. The two
containers are toggled by `set_mode`; exactly one is visible at a time.

### 3.2 Script API

```gdscript
class_name Hud
extends Control

enum Mode { LEVEL, OVERWORLD }

const HEART_FULL  := preload("res://assets/sprites/hud/heart_full.png")
const HEART_EMPTY := preload("res://assets/sprites/hud/heart_empty.png")
const AMMO_FULL   := preload("res://assets/sprites/hud/ammo_full.png")
const AMMO_EMPTY  := preload("res://assets/sprites/hud/ammo_empty.png")
const ICON_SIZE   := 16

func set_mode(m: Mode) -> void
func set_health(hp: int, max_hp: int) -> void
func set_ammo(ammo: int, max_ammo: int) -> void
func set_score(score: int) -> void
func set_cleared(cleared: int, total: int) -> void
```

- `set_health(hp, max_hp)`: rebuild `HeartsRow` — clear children, add `max_hp`
  `TextureRect`s, each `HEART_FULL` if its index `< hp` else `HEART_EMPTY`.
- `set_ammo(ammo, max_ammo)`: rebuild `AmmoRow` symmetrically with `AMMO_FULL` /
  `AMMO_EMPTY`.
- `set_score(score)`: `ScoreLabel.text = "Score %d" % score`.
- `set_cleared(cleared, total)`: `ClearedLabel.text = "Levels cleared: %d / %d"`.
- `set_mode(m)`: show `LevelContainer` iff `m == LEVEL`, else show `OverworldContainer`.

Each `TextureRect` gets `custom_minimum_size = Vector2(ICON_SIZE, ICON_SIZE)` and
`expand_mode = IGNORE_SIZE` so the 16×16 source renders at a fixed cell. Rebuilding
rows on each setter call is cheap (≤ a handful of icons, infrequent updates).

### 3.3 Styling

Default Godot font with inline theme overrides on the labels (font size + color),
no `.theme` resource file. This keeps the change localized and avoids a new
asset type. A real sprite-font / theme pass is a later art task.

## 4. Icon assets

**Files:** `tools/gen_hud_icons.gd` (create), `assets/sprites/hud/*.png` (generated),
`assets/sprites/hud/LICENSE.txt` (create).

`gen_hud_icons.gd` extends `SceneTree`, run headless
(`godot --headless --path . --script res://tools/gen_hud_icons.gd`). It writes four
16×16 PNGs via `Image` + `save_png`:

| File | Content |
|---|---|
| `heart_full.png` | solid heart silhouette, bright red |
| `heart_empty.png` | heart outline only, dim |
| `ammo_full.png` | raygun-bolt glyph, bright yellow |
| `ammo_empty.png` | bolt outline only, dim |

All pixels are programmatically drawn (no external assets). Output is original/CC0;
`LICENSE.txt` mirrors the `assets/audio/LICENSE.txt` notice (generated placeholders,
public domain, replace freely). Artifacts are committed; the tool is re-runnable and
idempotent (overwrites). A `.import`/`.uid` sidecar pass is produced by `make import`
after generation.

## 5. Gamepad mapping

**Files:** `src/core/game_manager.gd` (modify `_ensure_input_actions`).

`_add_key_action(name, keycode)` is replaced by a helper that attaches a list of
`InputEvent`s to a (possibly new) action. The `if InputMap.has_action(name): return`
guard is preserved at the action level for idempotency. Per action:

| Action | Keyboard | D-pad button | Left-stick axis | Face button |
|---|---|---|---|---|
| `move_left`  | A (physical) | `JOY_BUTTON_DPAD_LEFT`  | `JOY_AXIS_LEFT_X`, value −1 | — |
| `move_right` | D (physical) | `JOY_BUTTON_DPAD_RIGHT` | `JOY_AXIS_LEFT_X`, value +1 | — |
| `move_up`    | W (physical) | `JOY_BUTTON_DPAD_UP`    | `JOY_AXIS_LEFT_Y`, value −1 | — |
| `move_down`  | S (physical) | `JOY_BUTTON_DPAD_DOWN`  | `JOY_AXIS_LEFT_Y`, value +1 | — |
| `jump`       | Space        | — | — | `JOY_BUTTON_A` (0) |
| `pogo`       | P            | — | — | `JOY_BUTTON_B` (1) |
| `shoot`      | X            | — | — | `JOY_BUTTON_X` (2) |
| `interact`   | Up           | — | — | `JOY_BUTTON_Y` (3) |

Axis events are `InputEventJoypadMotion` with `deadzone = 0.5`. Button events are
`InputEventJoypadButton`. No action is renamed or removed; existing keyboard
behavior is unchanged. No deadzone tuning UI, no rebinding.

> **Note on interact:** the existing hardcoded `KEY_UP` for `interact` is kept as-is;
this plan only *adds* `JOY_BUTTON_Y`. Generalizing the interact seam (making Up a
real action bind rather than a special case) is explicitly deferred.

## 6. Runtime wiring

**Files:** `src/runtime/level_runtime.gd` (modify `_build_hud`, `_on_level_completed`),
`src/runtime/player/player.gd` (add `max_health`).

### 6.1 `_build_hud(p)` rewrite

Replace the inline-`Label` body with:

1. Create `CanvasLayer` (name `"HUD"`, `layer = 1`), add to `LevelRuntime`.
2. Instantiate `res://src/ui/hud.tscn`, add to the layer.
3. Branch on `_level.map_kind`:
   - **LEVEL:** `hud.set_mode(Hud.Mode.LEVEL)`; seed
     `set_health(p.health, p.max_health)`, `set_ammo(p.ammo, p.max_ammo)`,
     `set_score(p.score)`; connect `p.score_changed`/`ammo_changed`/`health_changed`
     to the matching setters (reading `max_*` off the player at call time).
   - **OVERWORLD:** `hud.set_mode(Hud.Mode.OVERWORLD)`; compute
     `M = count of level.entities whose properties.target_level_id != ""`;
     `hud.set_cleared(GameManager.completed_levels.size(), M)`. No player signals.
4. Return early (no HUD) only if a future map_kind wants none — both LEVEL and
   OVERWORLD now get a HUD.

### 6.2 Player change

Add `@export var max_health: int = 3` to `Player` (default matches the existing
fixed `health = 3`). `take_damage` already clamps via `health -= amount` + death at
`<= 0`; no further clamping needed for the HUD (it displays `min(hp, max_hp)`-style
icons, and `health` never exceeds `max_health` in current gameplay). Hearts row uses
`max_health` slots.

### 6.3 Completion overlay layering

In `_on_level_completed`, set the overlay `CanvasLayer.layer = 10` (was implicit 0)
so it sits above the HUD (layer 1). One-line addition.

GUT, headless, deterministic. No visual/audio-output assertion (consistent with 6a).

- `test_jump_has_gamepad_button`: `InputMap.action_get_events("jump")` contains an

  `InputEventJoypadMotion` with `axis == JOY_AXIS_LEFT_X` and `axis_value < 0`.
